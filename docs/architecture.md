# Architecture

`ps-spa` has three layers:

1. `src/PsSpa`
   This is the framework core. It contains the HTML DSL, page model, request helpers, runtime, and browser bridge.

2. `scripts/cli`
   This is the code generator and project tooling. It owns route scanning, naming rules, code generation, quality artifacts, Tailwind scaffolding, and app scaffolding.

3. `examples/*`
   These are standalone example apps generated by the same tooling. They are not part of the framework core API.

Scaffolded apps own two app-level modules that the framework does not regenerate: `src/Shared.purs` (the shared-state record) and `src/Auth.purs` (the default `User` type + protect helpers). See [Getting Started](getting-started.md#shared-state-and-auth) for the contract. Edit them freely — `gen` and `verify` deliberately leave them alone, mirroring how `Shared.elm` / `Auth.elm` work in `elm-spa`.

## Core Modules

- `PsSpa.Page`
  Defines the four page kinds: `static`, `sandbox`, `element`, and `advanced`.

- `PsSpa.Runtime`
  Connects generated routes and generated pages to the browser runtime.

- `PsSpa.Html`
  Provides the JSX-like HTML DSL used by generated templates and user pages.

- `PsSpa.Request`
  Exposes page-facing request helpers for path, query, fragment, and raw href access.

- `PsSpa.Effect`
  Defines the page-output `Effect` that pages return from `init` / `update`. Includes `fromShared` for updating shared state from a page, which the runtime applies before re-rendering.

## Generator Modules

- `scripts/cli/routes.mjs`
  Turns files in `src/Pages` into route metadata.

- `scripts/cli/codegen.mjs`
  Generates `Generated.Route`, `Generated.Pages`, `Generated.App`, and `Generated.Link`.

- `scripts/cli/templates.mjs`
  Generates starter page modules for each page kind.

- `scripts/cli/scaffold.mjs`
  Generates a fresh app root for `ps-spa new`.

- `scripts/cli/quality.mjs`
  Generates smoke tests and benchmark scenarios.

## Build Layout

Each example app now builds to its own `.spago-output` directory. This avoids cross-project contamination when multiple apps contain modules with the same names such as `Main` or `Pages.Index`.
